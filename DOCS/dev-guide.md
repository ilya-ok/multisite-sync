# Руководство разработчика

## Быстрый старт

1. Прочитай `CLAUDE.md` (индекс) → выбери нужный файл в `DOCS/`
2. Ключевые файлы для большинства задач:
   - `includes/class-mps-bulk-edit.php` — редактирование цен
   - `includes/class-mps-copy.php` — копирование товаров
   - `includes/class-mps-sync.php` — синхронизация

## Частые задачи

| Задача | Куда идти |
|--------|-----------|
| Добавить поле в таблицу цен | `build_products_table()` в MS_Bulk_Edit + `ajax_save_product()` + `bulk-edit.js` |
| Добавить поле в копирование товаров | `ajax_copy_product()` в MS_Copy: добавить в `$data`, обработать в шаге 2 |
| Добавить поле в копирование страниц | `ajax_copy_page()` в MPS_Copy_Pages: добавить в `$data`, `update_post_meta` в шаге 2 |
| Изменить логику синхронизации | `sync_to_all_sites()` в MS_Sync |
| Добавить настройку | `render_settings_page()` + `save_settings()` в MS_Admin |
| Добавить мета-поле в синхронизацию | Массив `$meta_keys` в `sync_product_price()` и `ajax_save_product()` |

## Шаблон новой функции

```php
// 1. Добавить опцию (если нужно)
add_site_option('ms_new_option', '1');

// 2. Добавить метод в класс
public function new_feature() {
    // код
}

// 3. Добавить хук
add_action('hook_name', [$this, 'new_feature']);

// 4. Обновить DOCS/
```

## Отладка

- Включить `ms_debug_mode` в Network Admin → Настройки
- Проверить `wp-content/debug.log`
- Network Tab браузера: AJAX запросы `admin-ajax.php`
- PHP ошибки при копировании могут появиться в ответе AJAX (response.data)

## Важные ограничения

- **SKU обязателен** — без него синхронизация и копирование пропускают товар
- **Флаг `$is_syncing`** — предотвращает рекурсию при автосинхронизации
- **Лимит 200 товаров** на категорию в таблице (параметр `posts_per_page`)
- **Категории на дочерних сайтах** должны существовать заранее (копирование ищет по slug)
- **Taxonomy-атрибуты** (pa_*): термины создаются при копировании, но таксономия (`pa_color` и т.д.) должна быть зарегистрирована на дочерних сайтах
- **Таймаут AJAX копирования** — 120 сек на JS-стороне (sideload изображений может быть медленным)
